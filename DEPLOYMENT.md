# Deployment Guide - Option 1: Vercel + Railway + MongoDB Atlas

This guide walks you through deploying the Society Benchmarking System using:
- **Frontend**: Vercel (free tier available)
- **Backend**: Railway (free tier available)
- **Database**: MongoDB Atlas (free tier available)

## Total Setup Time: ~20-30 minutes

---

## Step 1: Setup MongoDB Atlas (Database)

### 1.1 Create MongoDB Atlas Account
1. Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)
2. Click "Try Free" or "Sign Up"
3. Create an account (email + password)

### 1.2 Create a Cluster
1. Click "Create" to build a new database
2. Select "M0 Sandbox" (FREE tier)
3. Select cloud provider: AWS
4. Select region: closest to your location (e.g., ap-south-1 for India)
5. Click "Create Cluster" (takes 1-3 minutes)

### 1.3 Create Database User
1. In left sidebar, go to "Database Access"
2. Click "Add New Database User"
3. Enter:
   - Username: `admin`
   - Password: Generate a strong password (copy it!)
   - Select "Autogenerated Secure Password"
4. Click "Add User"

### 1.4 Setup Network Access
1. Go to "Network Access" in left sidebar
2. Click "Add IP Address"
3. Select "Allow Access from Anywhere" (for development; restrict in production)
4. Click "Confirm"

### 1.5 Get Connection String
1. Go to "Clusters" → "Connect"
2. Click "Connect your application"
3. Select "Node.js" driver, version "5.0 or later"
4. Copy the connection string
5. **Replace `<password>` with your database password**

Example:
```
mongodb+srv://admin:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/society-comparison?retryWrites=true&w=majority
```

**Save this** - you'll need it for backend deployment!

## Step 2: Deploy Backend on Railway

### 2.1 Create Railway Account
1. Go to [Railway.app](https://railway.app)
2. Click "Start Project" 
3. Sign up with GitHub (recommended)
4. Authorize Railway to access your GitHub

### 2.2 Create New Project
1. Click "New Project"
2. Select "Deploy from GitHub repo"
3. Select your `societycomparison` repository
4. Click "Deploy"

### 2.3 Configure Environment Variables
1. Railway will auto-detect Node.js
2. Go to "Variables" tab
3. Add these variables:
   ```
   MONGODB_URI=mongodb+srv://admin:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/society-comparison?retryWrites=true&w=majority
   PORT=8000
   NODE_ENV=production
   ADMIN_PASSWORD=admin123
   ```
4. Click "Deploy"

### 2.4 Update CORS (Important!)
1. Once deployed, Railway will give you a URL like: `https://societycomparison-production-xxxx.up.railway.app`
2. Note this URL (you'll need it for frontend)
3. In your local code, update `server/index.js` CORS:
   ```javascript
   app.use(cors({
     origin: ['http://localhost:5173', 'https://your-vercel-frontend-url.vercel.app'],
     credentials: true,
   }));
   ```
4. Commit and push this change:
   ```bash
   git add server/index.js
   git commit -m "Update CORS for production deployment"
   git push origin main
   ```

### 2.5 Get Backend URL
- Go to Railway dashboard
- Click your project
- In "Public Networking", get your backend URL
- Save it for frontend configuration

---

## Step 3: Deploy Frontend on Vercel

### 3.1 Create Vercel Account
1. Go to [Vercel.com](https://vercel.com)
2. Click "Sign Up"
3. Sign up with GitHub (recommended)
4. Authorize Vercel to access your GitHub

### 3.2 Import Project
1. Click "Add New..." → "Project"
2. Select "Import Git Repository"
3. Paste your GitHub repo URL: `https://github.com/iaashishjindal/societycomparison`
4. Click "Continue"

### 3.3 Configure Project
1. Select "Next.js" or "Other" (it will auto-detect Vite)
2. **Root Directory**: `client/`
3. **Framework**: Vite
4. **Build Command**: `npm run build`
5. **Output Directory**: `dist`

### 3.4 Add Environment Variables
Before deploying, add environment variables:

1. Click "Environment Variables"
2. Add:
   ```
   VITE_API_URL=https://your-railway-backend-url.up.railway.app
   ```
   Replace with your actual Railway backend URL!

3. Click "Deploy"

### 3.5 Update Frontend API Configuration
1. Update `client/src/services/api.js`:
   ```javascript
   const API_URL = import.meta.env.VITE_API_URL || 'http://localhost:5000/api';
   ```

2. Commit and push:
   ```bash
   git add client/src/services/api.js
   git commit -m "Configure API URL for production"
   git push origin main
   ```

3. Vercel will auto-redeploy when you push

---

## Step 4: Update CORS Again (With Vercel URL)

After Vercel deploys, you'll have a URL like: `https://societycomparison.vercel.app`

### 4.1 Update Backend CORS
1. Update `server/index.js`:
   ```javascript
   app.use(cors({
     origin: [
       'http://localhost:5173',
       'https://societycomparison.vercel.app' // Your Vercel URL
     ],
     credentials: true,
   }));
   ```

2. Push to GitHub:
   ```bash
   git add server/index.js
   git commit -m "Update CORS with Vercel frontend URL"
   git push origin main
   ```

3. Railway will auto-redeploy

---

## Step 5: Verify Everything Works

### Test the Deployment
1. Go to your Vercel frontend URL: `https://your-project.vercel.app`
2. Check Home page - should load without errors
3. Go to Comparison page - should be empty initially
4. Go to Admin - login with password `admin123`
5. Add a test society and benchmarks
6. Go back to Comparison - should see your data

### Troubleshooting

**Frontend shows 404:**
- Clear browser cache (Cmd+Shift+R or Ctrl+Shift+R)
- Check Vercel deployment status

**Backend API calls fail:**
- Check `VITE_API_URL` environment variable in Vercel
- Ensure CORS is configured correctly
- Check Railway backend logs

**MongoDB connection fails:**
- Verify connection string in Railway `MONGODB_URI`
- Check MongoDB Atlas network access settings
- Ensure password is URL-encoded if it contains special characters

**Admin login not working:**
- Check `ADMIN_PASSWORD` in Railway environment variables
- Default is `admin123`
